# 通用语言 (Ubiquitous Language)

> 本文档定义了本项目的领域通用语言，包含中文术语和英文术语对照。
> 所有团队成员和代码中的命名应当以此为准。

---

## 分层架构 (Layered Architecture)

| 中文术语 | 英文术语 | 说明 |
|---------|---------|------|
| 领域层 | Domain Layer | 核心业务逻辑，不依赖任何其他层 |
| 应用层 | Application Layer | 用例编排，协调领域对象完成用户请求 |
| 基础设施层 | Infrastructure Layer | 技术实现，如持久化、外部服务 |
| 表现层 | Presentation Layer | 用户界面，Vue 组件和 Store |

---

## 领域实体 (Domain Entities)

### 文档 (Document) — 聚合根

| 中文术语 | 英文术语 | 说明 |
|---------|---------|------|
| 文档 | Document | 聚合根，包含一组问题和对应的回答 |
| 文档 ID | Document.id | 唯一标识 |
| 文档标题 | Document.title | 文档名称，不能为空 |
| 文档问题列表 | Document.questions | 文档关联的 Question 列表，用于导航 |
| 文档回答列表 | Document.answers | 文档关联的 Answer 列表 |
| 问题数量 | Document.questionCount | 问题的数量 |
| 创建时间 | Document.createdAt | 文档创建的时间戳 |
| 更新时间 | Document.updatedAt | 文档最后更新的时间戳 |
| 更新标题 | Document.updateTitle() | 修改文档标题，同时更新 updatedAt |
| 添加问题 | Document.addQuestion() | 向文档新增一个问题 |
| 移除问题 | Document.removeQuestion() | 从文档中删除指定问题 |
| 重排序问题 | Document.reorderQuestions() | 按指定 ID 顺序重新排列问题 |
| 获取摘要 | Document.getSummary() | 获取文档的概要信息 |
| 序列化 | Document.toJSON() | 将文档转为 JSON 格式 |
| 文档摘要 | DocumentSummary | 文档的轻量摘要（id, title, questionCount, answerCount） |

### 问题 (Question)

| 中文术语 | 英文术语 | 说明 |
|---------|---------|------|
| 问题 | Question | 文档的问题，用于组织结构和快速导航 |
| 问题 ID | Question.id | 唯一标识 |
| 问题文本 | Question.text | 问题的内容文字，作为导航标题显示 |
| 问题序号 | Question.order | 在文档中的排序位置，决定导航列表顺序 |
| 创建时间 | Question.createdAt | 问题创建的时间戳 |
| 更新时间 | Question.updatedAt | 问题最后更新的时间戳 |
| 更新文本 | Question.updateText() | 修改问题内容，同时更新 updatedAt |
| 变更序号 | Question.changeOrder() | 修改排序位置，同时更新 updatedAt |
| 序列化 | Question.toJSON() | 将问题转为 JSON 格式 |

### 回答 (Answer)

| 中文术语 | 英文术语 | 说明 |
|---------|---------|------|
| 回答 | Answer | 对某个问题的回复内容 |
| 回答 ID | Answer.id | 唯一标识 |
| 关联问题 ID | Answer.questionId | 所关联问题的 ID |
| 回答内容 | Answer.content | 回答的正文内容 |
| 创建时间 | Answer.createdAt | 回答创建的时间戳 |
| 更新时间 | Answer.updatedAt | 回答最后更新的时间戳 |
| 编辑内容 | Answer.editContent() | 修改回答内容，同时更新 updatedAt |
| 序列化 | Answer.toJSON() | 将回答转为 JSON 格式 |

| 中文术语 | 英文术语 | 说明 |

| 中文术语 | 英文术语 | 说明 |

## 领域事件 (Domain Events)

| 中文术语 | 英文术语 | 触发时机 |

## 持久化 (Persistence)

### 文档存储 (SQLite)

| 中文术语 | 英文术语 | 说明 |
|---------|---------|------|
| SQLite | SQLite | 嵌入式关系型数据库，应用内使用 better-sqlite3 访问 |
| 数据库文件 | Database File | `py-data/chat-manager.db`，位于项目根目录下 |
| 文档表 | documents table | 存储文档的 id、title、created_at、updated_at |
| 问题表 | questions table | 存储问题，通过 document_id 外键关联到文档，包含 text、sort_order、created_at、updated_at |
| 回答表 | answers table | 存储回答，通过 question_id 外键关联到问题（一对一），包含 content、created_at、updated_at |
| 事务 | Transaction | 确保文档保存/删除时关联数据的一致性 |
| 外键级联删除 | CASCADE DELETE | 删除文档时自动删除关联的问题和回答 |
| 预写式日志 | WAL (Write-Ahead Logging) | SQLite 的 WAL 模式，提升并发读写性能 |

### 设置存储 (LocalStorage)

| 中文术语 | 英文术语 | 说明 |
|---------|---------|------|
| 本地存储 | LocalStorage | 浏览器的本地存储 API，用于存储应用设置 |
| 设置键名 | Settings Key | `chat-manager-settings`，存储设置的 localStorage 键 |
| 默认设置 | Default Settings | 应用首次运行时的默认配置（如 darkMode: false） |

## Electron IPC

| 中文术语 | 英文术语 | 说明 |
|---------|---------|------|
| IPC 通道 | IPC Channel | 渲染进程与主进程之间的异步通信通道 |
| 主进程 | Main Process | Electron 主进程，负责数据库操作和系统级功能 |
| 渲染进程 | Renderer Process | 运行 Vue 应用的进程，通过 IPC 调用数据库操作 |
| 预加载脚本 | Preload Script | 通过 contextBridge 暴露 IPC 通道到渲染进程 |
