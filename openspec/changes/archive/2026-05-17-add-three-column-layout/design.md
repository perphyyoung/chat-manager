# 三栏布局设计

## 上下文

当前应用使用 Vue 3 + TypeScript + Vite 技术栈，需要实现一个三栏布局的主界面。用户需要直观地管理对话式文档，包括文档导航、对话内容展示和问题列表。

## 目标/非目标

### 目标

- 实现左中右三栏布局
- 左侧栏：文档列表，支持文档选择和导航
- 中间栏：对话式文档展示，问题右对齐，回答左对齐
- 右侧栏：当前文档的问题列表，支持快速跳转
- 响应式布局适配
- 平滑的用户交互体验

### 非目标

- 不涉及后端 API 开发
- 不涉及数据库设计
- 不涉及认证授权功能

## 决策

### 1. 布局方案

- 使用 Flexbox 实现三栏布局
- 左侧栏固定宽度 280px
- 右侧栏固定宽度 280px
- 中间栏自适应填充剩余空间
- 支持最小宽度响应式调整

### 2. 组件结构

```
src/components/
├── Layout/
│   └── ThreeColumnLayout.vue    # 三栏布局容器
├── Document/
│   ├── DocumentList.vue         # 左侧文档列表
│   └── DocumentItem.vue         # 文档列表项
├── Conversation/
│   └── ConversationView.vue     # 中间对话展示
└── Question/
    ├── QuestionList.vue         # 右侧问题列表
    └── QuestionItem.vue         # 问题列表项
```

### 3. 对话样式

- 问题消息：右对齐，背景色为主题色，文字白色
- 回答消息：左对齐，背景色为浅色，文字深色
- 使用圆角卡片样式
- 支持滚动自动到底部

### 4. 状态管理

- 使用 Pinia store 管理文档列表和当前选中状态
- 文档数据结构包含：id, title, questions[], conversation[]
- 问题列表与对话内容联动

## 风险/权衡

| 风险 | 缓解措施 |
|------|----------|
| 大量文档可能导致左侧栏性能问题 | 实现虚拟滚动或分页加载 |
| 对话内容过长导致性能问题 | 实现懒加载和虚拟滚动 |
| 响应式布局在小屏幕上体验不佳 | 设置最小宽度限制，支持移动端适配 |
| 问题列表与对话内容不同步 | 使用 Pinia 状态管理确保数据一致性 |
